# 네이버 블로그 — DebateHeat 소개 초안

**제목:** DebateHeat: 여러 AI 에이전트 토론을 아침 한 페이지로

**키워드:** AI, 멀티에이전트, 히트맵, 데이터시각화, 오픈소스, Next.js

---

## 들어가며

요즘 한 주제에 VC, PM, 윤리, 엔지니어 같은 **여러 페르소나**를 두고 LLM 토론을 돌리는 실험이 늘고 있습니다. 문제는 **로그가 너무 길다**는 것입니다. 결정과 논점이 묻힙니다.

**DebateHeat**는 하루치 토론 결과를 **구조화된 브리프**로 보여주는 오픈소스 대시보드입니다. 상위 아이디어, 히트맵, 그룹별 견해 차이를 한 번에 봅니다.

---

## 한 화면에 담기는 것

- 오늘의 **토픽**과 실행 메타데이터  
- **상위 아이디어** 목록과 점수  
- (이전 날이 있으면) **전일 대비 진전**  
- **히트맵 A:** 아이디어 × 평가축  
- **히트맵 B:** 키워드 동시출현  
- **히트맵 C:** 에이전트 그룹 × 아이디어  
- 상위 아이디어 **레이더** 차트  
- 그룹 간 **극단적 관점** 요약  

**아침에 스캔하기 좋은 순서**로 배치되어 있습니다.

---

## 동작 방식 (간단히)

앱은 날짜별 **JSON 파일**을 읽어서 그립니다. 실제 토론·채점은 별도 오케스트레이터(파이썬 등)에서 돌리고, 결과만 JSON으로 넣으면 됩니다. 저장소에는 **목(mock) 생성 스크립트**도 포함되어 있어 API 키 없이 UI를 시험할 수 있습니다.

---

## 이런 분께

- 멀티에이전트 브레인스토밍을 운영하는 개발자·리서처  
- 긴 생성 로그 대신 **요약 화면**이 필요한 팀  
- 사내/팀 내 **데일리 인사이트** 페이지를 만들고 싶은 분  

---

## 시작하기

저장소 클론 후 `npm install` → `npm run dev` → 브라우저에서 `/brief/latest` 로 접속. 자세한 절차는 문서의 **QUICKSTART**와 **튜토리얼**을 참고하세요.

**링크:** YOUR_REPO_OR_DEMO_URL

---

## 맺음말

멀티에이전트 시대에 부족한 것은 종종 **집계 UI**입니다. DebateHeat는 “하루 한 JSON, 아침 한 페이지”라는 단순한 계약을 제안합니다.

*본 프로젝트는 기본 제공 시 데이터 시각화에 초점이 있으며, 유료 LLM 토론 실행은 별도 연동이 필요합니다.*
